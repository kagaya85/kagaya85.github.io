进程间通信

1. 无名管道

  • test1-3中建立父子进程间建立两个管道才能实现双向传递数据
  • 无名管道传递的数据类型，长度是否有限制？ 数据类型为比特流，管道的容量有限制，不同系统不一样，当进程往管道里写入数据时，进程会阻塞
  • 能否在独立进程之间使用管道？ 不能，管道只能在父子进程和亲戚进程之间使用

2. 有名管道

  • test2-3中，双向传递数据仍需要建立两个不同的有名管道
  • 独立进程单向传递
  • 独立进程双向传递，双向传递数据需要建立两个有名管道
  • 有名管道传递的数据的类型，长度是否有限制？ 传递的数据类型为比特流，系统对任意时刻在一个FIFO中可以存在的而数据长度是有限制的，由#define PIPE_BUF语句定义，在Linux和许多其他类UNIX 系统中，它的值通常是4096 字节
  • 和无名管道的区别 无名管道由于管道名是默认的，只能用于亲戚进程之间的通信，而有名管道可以指定管道的名字，用于在不同独立进程之间通信。

  • 双向传递数据 4-2-1 4-2-2

  • 消息队列方式的数据类型，长度是否有限制？ 消息的数据结构却有一定的要求，指针msg_ptr所指向的消息结构一定要是以一个长整型成员变量开始的结构体，接收函数将用这个成员来确定消息的类型。 比如结构体定义如下：

    struct Message {
    long int msg_type;
    char text[BUFSIZE];
    };

Linux系统中有两个宏定义: MSGMAX, 以字节为单位，定义了一条消息的最大长度。 MSGMNB, 以字节为单位，定义了一个队列的最大长度。

  • 消息队列方式和有名/无名管道的区别
  • 消息队列可以实现全双工通信，而管道只能实现半双工通信
  • 消息队列可以通过设置消息类型来几乎完全避免命名管道的同步和阻塞问题
  • 消息队列可以用一些方法来提前查看紧急消息

6.Unix套接字方式

  • 阻塞方式 6-1-1 6-1-2 阻塞方式下写入极限
  • Unix类型Socket与TCP Socket相比有什么不同？
  • Unix Socket需要将socket的地址绑定到文件，包含普通路径名和抽象路径名
  • 对于Unix Socket来说，int socket (int domain, int type, int protocol)函数的第二个参数SOCK_STREAM和SOCK_DGRAM都是可靠的，第三个参数一定设置为0

  • UNIX域套接字的地址由sockaddr_un结构表示，sun_family设置为AF_UNIX

    struct sockaddr_un {
    sa_family sun_family; / AF_UNIX /
    char sun_path[108]; / pathname /
    };

  • 非阻塞方式
  • 有非阻塞方式，设置方法同TCP Socket
  • 可以使用select来控制读写
  • 当在写之前不使用select，写满后直接返回负值
  • 使用select后，无法写入时阻塞
