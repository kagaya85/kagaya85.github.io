Vue.js学习笔记 part 1

在学校期间重新认识了JavaScript这门动态语言的强大功能，尤其是在当今H5越来越主流的趋势下JavaScript也在大放异彩，于是寒假开始打算捡起好久没用的JavaScript，一面重新回顾一下基础语法，另一面打算接触一款前端框架，利用这个假期的时间好好研究一下。 了解了下目前前端最热门的框架比如Facebook的React以及Vue.js，在询问了朋友的意见之后最终还是决定从Vue入手。接下来便决定一边研读Vue的官方文档，一边用学习笔记的方式记录学习过程，并尽量用简单的语言表达出来，方便之后查看。（不得不说无论是Vue还是React，之所以受到广大开发者的欢迎除了其易用性外，详尽的多语言官方文档也是必不可少的） 准备

  • 如何使用Vue.js

  • 可以直接在html页面中引入用于开发环境的Vue库文件，包含有详尽的命令行警告等调试信息

  • 或使用用于生产环境，优化了库的大小和加载速度的Vue库

  • 如果有安装Node.js可以在开发目录下使用本地安装Vue库，之后会将Vue相关文件安装到运行目录下的node_modules目录下，使用时可通过require引入

    npm install vue

  • 当然还有使用Vue官方的开发环境配置工具vue-cli，如果不熟悉使用Node.js的构建工具的话暂时还是不建议使用

认识Vue.js

  • Vue的核心——声明式渲染

Vue可以利用简洁的模板语法来声明式地将数据在渲染地时候加入到DOM系统中，这点似乎有点类似XAML里绑定数据更新源地操作，但对于脚本语言来说似乎更加方便直接，很好的将UI代码和逻辑代码实现了分离。 举一个官方的例子 HTML：

<div id="app">
    { { message }}</div>

JavaScript：

var app = new Vue({
el: '#app',
data: {
message: 'Hello Vue!'
}
})

el表示element，看来这个选择器是可以利用 css选择器的语法的，data大概是为了绑定相关的数据，这里将message的值渲染为'Hello Vue!'

  • 除了文本插值又或者可以使用Vue的特性——指令

HTML:

<div id="app-2">
    <span v-bind:title="message">
鼠标悬停几秒钟查看此处动态绑定的提示信息！
</span></div>

JavaScript:

var app2 = new Vue({
el: '#app-2',
data: {
message: '页面加载于 ' + new Date().toLocaleString()
}
})

这里的v-前缀表示将span的title属性与代码实例中的message保持一致 以上的值也可以直接通过浏览器中的JavaScript控制台来使用命令直接操作，如：

app.message = '新消息'

同时便可以直接的在浏览器中看到属性值的变化，可见Vue在后台将数据和DOM已经做了直接的关联，可以轻松实现响应式的网页 Vue指令 ​ 除了上文提到的v-bind下面会补充几个常用的Vue指令

  • 条件显示：v-if

HTML：

<div id="app-3">
<p v-if="seen">现在你看到我了</p>


</div>

JavaScript：

var app3 = new Vue({
el: '#app-3',
data: {
seen: true
}
})

在控制台输入app3.seen = false，对应的p标签就变得不可见了 Vue有强大的过渡效果系统，可以在插入/更新/删除元素时自动应用过渡效果

  • for循环：v-for

HTML：

<div id="app-4">
<ol>
    <li v-for="todo in todos">
        { { todo.text }}</li>
</ol>
</div>

JavaScript：

var app4 = new Vue({
el: '#app-4',
data: {
todos: [
{ text: '学习 JavaScript' },
{ text: '学习 Vue' },
{ text: '整个项目' }
]
}
})

将数组元素中的每一项循环生成了对应的li标签，NB！ 甚至可以再在控制台中使用app4.todos.push({ text: '新项目' })来添加新的项目，或使用pop方法来删去项目

  • 事件监听：v-on

HTML：

<div id="app-5">

{ { message }}

<button v-on:click="reverseMessage">逆转消息</button>

</div>

JavaScript：

var app5 = new Vue({
el: '#app-5',
data: {
message: 'Hello Vue.js!'
},
methods: {
reverseMessage: function () {
this.message = this.message.split('').reverse().join('')
}
}
})

这里出现了新的methods关键字，将所有绑定的函数放在这里，例子中将button标签的click事件绑定到了reverseMessage而这个属性是一个匿名函数，用于将该对象中的message字符串做反转，有点好奇这里的split()函数按空格划分后在调用recerse()难道不是只将数组元素调换位置，得到Vue.js! Hello嘛，而是对每一项字符串也做了reverse操作，先记一下。

  • 状态-输入双向绑定：v-model

HTML：

<div id="app-6">

{ { message }}

<input v-model="message">

</div>

JavaScript：

var app6 = new Vue({
el: '#app-6',
data: {
message: 'Hello Vue!'
}
})

上面示例可以直接将input输入的值通过message及时反馈到p标签的内容上，轻松实现双向的数据绑定，舒服 Vue组件 ​

  • 注册组件

组件是可复用的 Vue 实例，且带有一个名字，这里为todo-item

// 定义名为 todo-item 的新组件
Vue.component('todo-item', {
// todo-item 组件现在接受一个
// "prop"，类似于一个自定义特性。
// 这个 prop 名为 todo。
props: ['todo'],
template: '
    <li>{ { todo.text }}</li>
'
})

这里的props: ['todo']不是很理解为什么这样写，为什么不是类似data：的形式，目测component作为Vue的一个子对象，需要通过这中方式传值，先记一下，不过明白这里需要接受todo的传值运用于模板中

  • 构建组件模板






将有序列表中的li替换为一个自定义标签todo-item，对应前面定义的组件，这里绑定todo到对应List中的每一个item，传递给模板后按模板显示出todo.text 剩余JavaScript代码：

var app7 = new Vue({
el: '#app-7',
data: {
groceryList: [
{ id: 0, text: '蔬菜' },
{ id: 1, text: '奶酪' },
{ id: 2, text: '随便其它什么人吃的东西' }
]
}
})

这样可以为模板增加更加复杂的逻辑而不会影响到其父单元的调用，实现了很好的界面与逻辑代码的分离
