Vue.js学习笔记 part 2

Vue实例

  • 每个Vue应用都是通过Vue函数创建一个新的Vue实例

    var vm new Vue({
    // 选项
    })

Vue的设计收到了MVVM（Model-View-ViewModel）的启发，所以常用vm（ViewModel的缩写）来表示Vue实例

  • Object.freeze()

阻止修改现有属性，使响应系统无法追踪变化 JavaScript：

var obj = {
foo: 'bar'
}

Object.freeze(obj)

new Vue({
el: '#app',
data: obj
})

HTML：

<div id="app">

{ { foo }}

<!-- 这里的 `foo` 不会更新！ -->
<button>Change it</button>

</div>
  • Vue实例属性方法

以$为前缀，以便与用户定义的属性区分 JavaScript：

var data = { a: 1 }
var vm = new Vue({
el: '#example',
data: data
})

vm.$data === data // =&gt; true
vm.$el === document.getElementById('example') // =&gt; true

// $watch 是一个实例方法
vm.$watch('a', function (newValue, oldValue) {
// 这个回调将在 `vm.a` 改变后调用
})

这里的data、el都是该Vue实例vm的属性，watch为实例的方法，可以建立一个监视函数，当a的值发生改变时调用该函数 实例的生命周期钩子

  • Vue实例的生命周期

  • 钩子

可以用来建立在Vue实例生命周期的特定时刻执行的函数

  • created钩子用来在一个实例被创建以后执行代码

JavaScript：

new Vue({
data: {
a: 1
},
created: function () {
// `this` 指向 vm 实例
console.log('a is: ' + this.a)
}
})
// =&gt; "a is: 1"

其他钩子的位置如图中红框所示 ⚠️注意：

不能在选项属性或者回调函数上使用箭头函数 如：created: () =&gt; console.log(this.a) 或：vm.$watch('a', newValue =&gt; this.myMethod())

原因是箭头函数是与父级的上下文绑定在一起的，导致this指针不指向Vue实例
