Vue.js学习笔记 part 3

插值

  • 文本

使用Mustache语法，双大括号表示文本插值

<span>Message: { { msg }}</span>

当Vue对象中的msg属性改变时，插值处的内容也都会更新 v-once指令表示一次性插值，之后对应数据改变时，插值处内容也不会更新（可能会影响到该节点上其他的数据绑定）

<span v-once="">这个将不会改变: { { msg }}</span>
  • 原始HTML

v-html指令输出可执行的HTML代码，而非普通文本

Using mustaches: { { rawHtml }}

Using v-html directive: <span v-html="rawHtml"></span>

⚠️注意：只需对可信内容使用HTML插值，绝不要对用户提供的内容使用，防止XSS攻击（Cross-site scripting） 特性 v-bind可以将Vue对象的属性绑定到HTML的特性上

<div v-bind:id="dynamicId"></div>

若属性为布尔值（如disabled），则v-bind效果略有不同

<button v-bind:disabled="isButtonDisabled">
Button
</button>

当值为null、undefined或false，则disabled特性不会出现在渲染出来的<button>元素中 JavaScript表达式 队友绑定的数据可以使用JS表达式

{ { number + 1 }}

{ { ok ? 'YES' : 'NO' }}

{ { message.split('').reverse().join('') }}
<div v-bind:id="'list-' + id"></div>

绑定只限单个表达式，语句和流程控制都不能生效 缩写 v-bind HTML：

<!-- 完整语法 -->
<a v-bind:href="url">...</a>

<!-- 缩写 -->
<a :href="url">...</a>

v-on HTML：

<!-- 完整语法 -->
<a v-on:click="doSomething">...</a>

<!-- 缩写 -->
<a @click="doSomething">...</a>

计算属性 ​ computed

  • 举例

HTML：

<div id="example">

Original message: "{ { message }}"

Computed reversed message: "{ { reversedMessage }}"

</div>

JavaScript：

var vm = new Vue({
el: '#example',
data: {
message: 'Hello'
},
computed: {
// 计算属性的 getter
reversedMessage: function () {
// `this` 指向 vm 实例
return this.message.split('').reverse().join('')
}
}
})
  • 计算属性与方法

同样的操作也可以通过方法methods来实现 HTML：

Reversed message: "{ { reversedMessage() }}"

JavaScript：

// 在组件中
methods: {
reversedMessage: function () {
return this.message.split('').reverse().join('')
}
}

两种形式可以实现同样的效果，但是不同的是：

  • 计算属性基于依赖来进行缓存的，简单的说，只要依赖的值没有改变，就不需要重新计算，直接返回上次计算的缓存值
  • 方法在每次调用的时候都会重新执行一遍定义的函数，只用触发重新渲染，就会重新执行。

⚠️注意：若不是响应式的依赖，则不会触发计算属性的更新，如：

computed: {
now: function () {
return Date.now()
}
}

若不希望有缓存，则可以使用方法属性methods来代替计算属性 侦听器 ​ watch ​ 通过 watch 选项提供了一个更通用的方法，来响应数据的变化。当需要在数据变化时执行异步或开销较大的操作时，这个方式是最有用的
